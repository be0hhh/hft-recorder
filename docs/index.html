0. Что должно быть у пользователя

Пользователь НЕ компилирует проект. Ему нужен готовый архив примерно такого вида:

hft-recorder-package/
  hft-recorder-gui.exe          # Windows
  # или:
  hft-recorder-gui              # Linux

  recordings/
    1776994215_binance_futures_usd_RAVEUSDT/
      trades.jsonl
      # optional: bookticker.jsonl, depth.jsonl, snapshot_000.json

  start-windows.bat             # optional helper
  send-marker.ps1               # optional helper
  send-marker.sh                # optional helper
Главное: готовый GUI-бинарь + данные для Viewer. Никаких cmake, compile.sh, CXETCPP, headers или исходников пользователю не нужно.

1. Что REST Marker API реально делает

REST Marker API получает timestamp в nanoseconds и optional label, затем рисует вертикальную линию на уже открытом активном графике. 

external script / service
  -> POST http://127.0.0.1:18080/api/v1/chart/markers/vertical
  -> hft-recorder GUI
  -> yellow vertical line on active chart
REST API не загружает файл, не открывает график, не отправляет ордер, не симулирует биржу и не создаёт execution events. Это только визуальная отметка на уже открытом графике.

2. Как подготовить папку для человека

Самый простой вариант — один zip-архив:

hft-recorder-offline.zip
  hft-recorder-gui.exe
  recordings/
  start-windows.bat
  send-marker.ps1
  README.txt

README.txt можно сделать таким:

1. Распакуй архив в любую папку, например C:\hft-recorder-offline.
2. Запусти start-windows.bat.
3. В GUI открой Viewer.
4. Нажми Load / Open и выбери recordings\...\trades.jsonl или session folder.
5. Проверь REST:
   PowerShell: Invoke-RestMethod http://127.0.0.1:18080/api/v1/health
6. Отправь marker:
   .\send-marker.ps1 1776994215000000000 "my signal"

3. Как запустить готовый бинарь

Windows: двойной клик или PowerShell

Если бинарь уже собран как offline viewer, можно запускать напрямую:

cd C:\hft-recorder-offline

$env:HFTREC_API_HOST="127.0.0.1"
$env:HFTREC_API_PORT="18080"
$env:HFTREC_METRICS_PORT="8080"

.\hft-recorder-gui.exe

Windows helper: start-windows.bat

Этот файл можно положить рядом с hft-recorder-gui.exe, чтобы пользователь просто нажал два раза:

@echo off
set HFTREC_API_HOST=127.0.0.1
set HFTREC_API_PORT=18080
set HFTREC_METRICS_PORT=8080

cd /d "%~dp0"
start "" "%~dp0hft-recorder-gui.exe"

Linux

cd ~/hft-recorder-offline
chmod +x ./hft-recorder-gui

HFTREC_API_HOST=127.0.0.1 \
HFTREC_API_PORT=18080 \
HFTREC_METRICS_PORT=8080 \
./hft-recorder-gui
HFTREC_API_HOST=127.0.0.1 значит: REST доступен только на этой же машине. Это самый безопасный режим по умолчанию.

4. Как открыть график

1
Открыть GUI.

Должно появиться окно hft-recorder.

2
Перейти в Viewer.

REST marker рисуется только в Viewer / active chart.

3
Загрузить данные.

Выбрать session folder или конкретный trades.jsonl.

4
Убедиться, что график виден.

Если свечи/точки/линия не отображаются, REST marker тоже некуда рисовать.

Пример session folder:
C:\hft-recorder-offline\recordings\1776994215_binance_futures_usd_RAVEUSDT

Пример trades file:
C:\hft-recorder-offline\recordings\1776994215_binance_futures_usd_RAVEUSDT\trades.jsonl

5. Проверить, что REST видит график

PowerShell

Invoke-RestMethod http://127.0.0.1:18080/api/v1/health

curl

curl http://127.0.0.1:18080/api/v1/health

Нормальный ответ:

{"ok":true,"active_chart":true,"loaded":true}
ПолеЧто значитЧто делать, если false
okREST server отвечаетПроверить, запущен ли GUI и порт 18080
active_chartGUI нашёл активный ChartControllerПерейти в Viewer / открыть chart
loadedВ Viewer загружены данныеЗагрузить session folder или trades.jsonl

6. Отправить вертикальную линию на график

Endpoint:

POST http://127.0.0.1:18080/api/v1/chart/markers/vertical
Content-Type: application/json

{"ts_ns":"1776994215000000000","label":"signal A"}

PowerShell: one-liner

$body = @{ ts_ns = "1776994215000000000"; label = "signal A" } | ConvertTo-Json -Compress
Invoke-RestMethod -Method Post -Uri http://127.0.0.1:18080/api/v1/chart/markers/vertical -ContentType 'application/json' -Body $body

PowerShell helper: send-marker.ps1

param(
  [Parameter(Mandatory=$true)][string]$TsNs,
  [string]$Label = "manual signal"
)

$body = @{ ts_ns = $TsNs; label = $Label } | ConvertTo-Json -Compress

Invoke-RestMethod `
  -Method Post `
  -Uri "http://127.0.0.1:18080/api/v1/chart/markers/vertical" `
  -ContentType "application/json" `
  -Body $body

Использование:

.\send-marker.ps1 1776994215000000000 "my signal"

curl

curl -X POST http://127.0.0.1:18080/api/v1/chart/markers/vertical ^
  -H "Content-Type: application/json" ^
  --data "{\"ts_ns\":\"1776994215000000000\",\"label\":\"curl signal\"}"

Python

import requests

requests.post(
    "http://127.0.0.1:18080/api/v1/chart/markers/vertical",
    json={"ts_ns": "1776994215000000000", "label": "python signal"},
    timeout=1.0,
).raise_for_status()

Java 11+

import java.net.URI;
import java.net.http.HttpClient;
import java.net.http.HttpRequest;
import java.net.http.HttpResponse;

public class MarkerTest {
  public static void main(String[] args) throws Exception {
    String json = "{\"ts_ns\":\"1776994215000000000\",\"label\":\"java signal\"}";

    HttpRequest request = HttpRequest.newBuilder()
      .uri(URI.create("http://127.0.0.1:18080/api/v1/chart/markers/vertical"))
      .header("Content-Type", "application/json")
      .POST(HttpRequest.BodyPublishers.ofString(json))
      .build();

    HttpResponse<String> response = HttpClient.newHttpClient()
      .send(request, HttpResponse.BodyHandlers.ofString());

    System.out.println(response.statusCode());
    System.out.println(response.body());
  }
}

Очистить markers

Invoke-RestMethod -Method Delete http://127.0.0.1:18080/api/v1/chart/markers

7. Как выбрать правильный ts_ns

ts_ns должен попадать во временной диапазон загруженного графика. Если marker отправился, но не виден, чаще всего timestamp вне диапазона экрана или вне данных.

Если есть Unix seconds

# seconds -> nanoseconds
1776994215 * 1000000000 = 1776994215000000000

Python conversion

import datetime as dt

ts = dt.datetime.fromisoformat("2026-04-23T12:10:15+00:00")
ts_ns = int(ts.timestamp() * 1_000_000_000)
print(str(ts_ns))

Почему строкой?

Отправляй ts_ns строкой: "1776994215000000000". Большие int64 timestamps могут терять точность в JavaScript/JSON tooling.

8. Как отправлять “из любого места”

Та же машина

http://127.0.0.1:18080

Самый простой и безопасный вариант.

Docker на этой машине

http://host.docker.internal:18080

Если script/bridge внутри Docker Desktop.

Другая машина в LAN

HFTREC_API_HOST=0.0.0.0
HFTREC_API_PORT=18080

Потом обращаться к IP компьютера с GUI.

Пример: GUI на 192.168.1.50

curl -X POST http://192.168.1.50:18080/api/v1/chart/markers/vertical \
  -H 'Content-Type: application/json' \
  --data '{"ts_ns":"1776994215000000000","label":"remote signal"}'
Не открывай 0.0.0.0:18080 в публичный интернет. Это debug/offline API без нормальной auth-модели. Для LAN лучше закрыть порт firewall-ом только на доверенные IP.

9. Тест прямо с этой GitHub Pages страницы

Эта форма пробует отправить marker из браузера. Если будет CORS error, это нормально: используй PowerShell/curl/Python или добавь CORS headers в REST server.

Сначала запусти GUI, открой Viewer и загрузи trades/session.
